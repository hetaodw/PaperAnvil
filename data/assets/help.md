# PaperAnvil 使用指南与方法论说明

欢迎使用 PaperAnvil 自动化科研仪表盘。本项目集成了从定量统计、机器学习到 NLP 语义挖掘的全链路分析模型。

---

## 🛠️ 快速上手：API Key 申请指南
本项目核心驱动力来自 **阿里云“百炼” (DashScope)** 平台。

### 1. 账号注册与登录
- 访问：[阿里云百炼控制台](https://bailian.console.aliyun.com/)
- 使用阿里云账号登录（支持支付宝/钉钉扫码）。

### 2. 开通模型服务
- 在左侧菜单栏选择 **“模型广场”**。
- 搜索并确保已开通 `qwen-max` (文本核心) 和 `qwen-image-2.0` (绘图核心) 的服务权限。默认开启

### 3. 获取 API Key
- 点击左侧 **“API-KEY”** 菜单。
- 点击 **“创建我的 API-KEY”**。
- **重要建议**：请将生成的 `sk-xxxx` 密钥妥善保存，切勿泄露。

## 📊 数据分析方法论及其意义

### 1. 基础统计模块 (BasicStatsTool)
- **定位**：初识数据，建立宏观背景。
- **核心方法**：描述性统计。
- **实际意义**：
    - **人口学分布**：明确“谁在发声”，验证样本代表性。
    - **李克特量表**：均值反映普遍态度，标准差衡量人心分歧，分布图捕捉极少数极端意见。

### 2. 高级分析模块 (StateTool)
- **定位**：探索隐藏结构与核心驱动力。
- **核心方法**：机器学习。
- **实际意义**：
    - **智能分群 (Clustering)**：利用 K-Means 将用户标签化，实现精细化运营建议。
    - **异常检测**：剔除乱填样本或恶意反馈，清理数据噪声。
    - **相关性分析**：发现变量间的联动（如：年龄越大满意度越低）。
    - **核心因素评估**：利用随机森林评估哪个因素对“整体满意度”贡献最大（最高 ROI 指标）。

### 3. 语义分析模块 (SemanticTool)
- **定位**：听懂用户的“言外之意”。
- **核心方法**：NLP & LLM 定性分析。
- **实际意义**：
    - **主题建模**：将万千吐槽浓缩为几个核心痛点（化繁为简）。
    - **细粒度情感 (ABSA)**：精准定位用户是对“UI”不爽还是对“速度”不爽。
    - **语义聚类**：提炼最具有代表性的用户画像原声。

## 📂 数据架构说明：JSON 中间文件详解

系统运行过程中产生的中间数据存储在 `data/` 目录下，理解这些文件有助于深度复盘分析逻辑：

### 1. 过程定义类
- `questionnaire.json`: **调研基石**。定义了所有题目、选项及量表结构。
- `personas.json`: **受访原型**。记录了虚拟画像的性格背景及其在量表题上的随机倾向预设。

### 2. 统计结果类
- `basic_stats.json`: **描述报告**。存储了所有题目的频数、百分比、均值及标准差。
- `analysis_results.json`: **算法洞察**。包含 K-Means 聚类中心、孤立森林异常点明细、相关系数矩阵及特征重要性权重。
- `semantic_analysis.json`: **语义挖掘**。存储了 LDA 主题总结、ABSA 情感对以及基于向量空间的代表性原声。

### 3. 最终决策类
- `analysis_insights.json`: **LLM 最终裁判**。由 AI 结合上述所有 JSON 数据过滤噪声后生成的深度结构化推向，也是撰写最终报告的直接依据。

## 📝 常见问题 (FAQ)
- **Q: 为什么运行看板进度条停滞？**
    - A: 可能是网络波动，请检查 API Key 余额或网络连接稳定性。
- **Q: 报告生成缓慢？**
    - A: 系统正在进行海量数据扩增与深度分析，视样本量大小可能需要 3-10 分钟。
